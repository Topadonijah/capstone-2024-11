## Commit Convention

1. `feat`: **새로운 기능 추가**
2. `fix`: **버그 수정**
3. `docs`: **문서 수정**
4. `style`: **코드 포맷팅 → Code Convention**
5. `refactor`: **코드 리팩토링**
6. `test`: **테스트 코드**
7. `chore`: **빌드 업무 수정, 패키지 매니저 수정**
8. `comment`: **주석 추가 및 수정**

## Branch Naming Convention

브랜치를 새롭게 만들 때, 브랜치 이름은 항상 위 `Commit Convention`의 Header와 함께 작성되어야 합니다.

### 예시

> `feat/On_you`  
> `refactor/On_you`
